可以看到, 字符串的值被修改了 —— 这翻译成 C 语言的说法就是 : %eax 和 my_str 是指向同一个字符串的指针

所以用 mov 是不能传送字符串的

movs 指令

movs 系列指令的作用是把字符串从一个内存位置传送到另一个内存位置 —— 存在两个字符串副本

  • movsb : 传送单一字节 ( 1 字节, 8bit )
  • movsw : 传送一个字 ( 2 字节, 16bit )
  • movsl : 传送一个双字 ( 4 字节, 32bit )

Intel 文档使用 movsd 传送双字, GNU 汇编器决定使用 movsl

movs 指令使用隐含的源和目的操作数

  • 隐含的源操作数是 esi 寄存器, 指向源字符串的内存位置 —— s 代表 source
  • 隐含的目的操作数是 edi 寄存器, 指向字符串要被复制到的目的内存位置 —— d 代表 destination
